# Godot Asset Library submission receipt

Asset Library submission is deliberately manual because the library reviews every asset and requires public repository/release identity.

For version `0.1.0`:

1. Download the attested `godot-mcp-addon-0.1.0.zip`, `release-manifest.json`, and `SHA256SUMS` from the GitHub release.
2. Verify the GitHub artifact attestation and the archive SHA-256 against both files.
3. Confirm the archive contains `addons/godot_mcp/plugin.cfg`, the addon-local `README.md`, and `LICENSE` and contains no release-exported game files or credentials.
4. Submit the immutable GitHub tag/archive URL, Godot 4.7 compatibility, MIT license, tool category, and a square PNG/JPG icon of at least 128×128 through the Asset Library portal.
5. Record the Asset Library edit URL, submission timestamp, reviewer state, GitHub release URL, archive SHA-256, and submitter identity in `release/receipts/asset-library-0.1.0.json`.

Do not create the receipt until the real submission exists. A draft, rejected, or pending review is not a published Asset Library release.
